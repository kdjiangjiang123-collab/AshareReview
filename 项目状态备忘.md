## 交易持仓联动逻辑（重要！）
- **买入+同步** → 自动加入持仓，加权平均算成本
- **加仓+同步** → 同只股票合并，均价重新计算
- **卖出+同步** → 自动从持仓扣除数量，卖光自动标记平仓
- **卖光后再买回+同步** → 复用旧持仓行，成本重置为新的买入价（不会重复创建）
- **删除买入交易** → 自动从持仓扣除，恢复仓位
- **删除卖出交易** → 自动恢复持仓（含已平仓的），成本基准不变
- **代码规范化** → 自动去掉 sz/sh/bj 前缀，6位数字校验防止代码/名称填反

## 2026-07-02 修复记录

### 1. 搜索股票代码错乱（`stocks.py` + `tencent_api.py`）
- **现象**：搜"平安银行"或"payh"，代码变成 `sz`，报错"应为6位数字"
- **根因**：腾讯搜索接口格式是 `市场~代码~名称~拼音~类型`（如 `sh~600519~贵州茅台~gzmt~GP-A`），旧代码把市场前缀当作代码、把代码当名字
- **修复**：
  - `tencent_api.py` `search_stock_tencent()` 重写解析，正确拆分字段，只收A股
  - `stocks.py` 新增 `normalize_code()` 统一去 sz/sh/bj 前缀，所有入口统一调用

### 2. 买入价需要手动输入（`mode_trade.py`）
- **现象**：每次录入交易都要手打成交价
- **修复**：搜股后自动拉腾讯实时行情填入成交价，显示"💰 现价 XX.XX"

### 3. 手动添加持仓代码前缀问题（`mode_trade.py`）
- **现象**：手动添加持仓没做代码规范化，sz 前缀会存进数据库
- **修复**：`quick_add` 表单也走 `normalize_code()` 统一处理

### 4. 新增K线图（`components.py` + `mode_trade.py`）
- `components.py`: `render_kline_chart()` — Plotly 交互蜡烛图，含 MA5/10/20 + 成交量柱
- `mode_trade.py`: `_kline_viewer()` — 持仓页底部选股看图，支持10-120天

### 5. 启动脚本乱码（`启动复盘助手.bat`）
- **现象**：`chcp 65001` 导致中文变成乱码，命令行把乱码当指令执行
- **修复**：去掉编码切换，提示改英文
